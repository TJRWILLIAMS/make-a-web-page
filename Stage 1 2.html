Stage 0
HTML

HTML is hyper text markup language, and is the basic language of documents on the web.

It is made up of text content (what is actually displayed), markup (the formatting), references to other documents (images, videos etc), and (hyper)links to other pages on the internet.

Tags

There are a number of different tags, that perform different functions on the HTML.

For instance, the b tag makes text bold, the em tag makes it italic.

Inline vs Block

Tags, and elements more generally (the complete unit made up of two tags and their contents), can either be inline or block. The distinction is relatively simple, with inline elements such as b, img etc. taking their place "in the line" of the text, whereas block elements take the place of a separate block on the page.

Stage 1.1
HMTL, CSS and the DOM

HMTL is the language used to classify and define the structure of a web page. It allows you to determine the types of element and their contents.

CSS is the language used to set the style of the page - the formatting, font etc. CSS provides additional information on top of your HTML, and is useful to avoid too much unnecesary duplication in your code.

The Document Object Model (DOM) is a standard protocol for interpreting the elements in HTML.

This video does a good job of explaining the three.

The developer console and boxes

The developer console is a useful tool in google chrome (and similar tools are available in other web browsers) that allows you to see the individual elements and the raw HTML code behind the page you're viewing.

You can also see the styles, the log of requests in the browser and other details of your browser's activity.

Through the developer console, it should be apparent that every object on a we page is contained within a rectangular box. This is crucial for web formatting and front end development.

Boxes are created using div tags, with each div tag layer representing a box.

This video does a good job of explaining how divs work.

Cleaning up the code

Using classes allows us to set formatting to specific elements on our page, and saves us from repeatedly typing out the same formatting for similar elements

CSS works in something like the reverse of the legal system - it works from the most precise rule (the lowest child element) up to the most general rule (that would likely apply to the whole page).

How to go about formatting your page

Trial and error appears to be the most effective way of formatting, especially when working towards a set design. You follow a simple process of formatting, testing and re-tweaking until the result is what would be desired.

The sheer bulk of information available for CSS means that it is not advisable to try and learn all the attributes that one might want to try and use, so instead using the documentation is highly recommended - such as the link below.

CSS Documentation

The most important thing when coding appears to be maintaining a proper and regular structure so as to make debugging and error checking more straightfoward. Furthermore, this will likely make it easier for other programmers to read and understand your work. Hence the need for logical class names, grouping similar information together and so forth.
